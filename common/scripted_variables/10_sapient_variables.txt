# --- Scaling steps (added per iteration until a stop gate trips) -----------
# --- Basic-resource SHOCK BUFFER ------------------------------------------
# Flat floors give the AI zero shock absorption: it budgets to current income,
# so a temporary windfall (e.g. the +200 energy a purge produces while grinding
# down a captured planet's pops) gets spent on RECURRING costs like fleet upkeep,
# and the economy craters the moment the windfall ends. Vanilla guards this with
# CAPPED basic-resource scaling lanes; we dropped those when the ladder was
# replaced, leaving bare floors. These restore them.
# Self-paced: the treadmill climbs ONE district at a time as each step is
# fulfilled, it does not jump to the cap. The cap then makes the buffer fade out
# on its own -- +200 is decisive early game, ~5-10% noise once income is in the
# thousands, which is exactly when the protection stops being needed.
# PER-RESOURCE caps: a resource is only worth buffering to the extent it's
# actually consumed. TESTED FINDING: a flat cap is implicitly sized for a
# MID-GAME economy — +200 energy is ~a whole second economy at 2205, so the
# treadmill ate the early build order filling a cushion the empire couldn't
# use. Energy / minerals / fleet-food caps are therefore TECH-TIERED (the same
# ladder idiom as the alloy stockpile stops): buffer +50 base, +100 at the
# zone-specialization tech, +200 one tier above. Ceilings are INCOME totals =
# base floor + buffer. Rungs live in 10_sapient_plan_triggers
# (sapient_*_buffer_over_cap).
#   energy   tiers: tech_power_hub_1 / tech_power_hub_2 (energy grid / nexus)
#   minerals tiers: tech_mineral_purification_1 / _2 (t2 = 250 = old flat cap)
#   fleet food tiers: tech_food_processing_1 / _2   (bioship base 60 = 25+35)
#   food(normal) 50 flat, UNTIERED: pure pop consumption, cushion already tiny
#   cons_gds 100 flat, UNTIERED for now (tier with
#     tech_consumer_good_refinement_1/2 if CG overbuild shows up early)
@sapient_energy_buffer_cap_t0 = 75    # 25 base + 50
@sapient_energy_buffer_cap_t1 = 125   # 25 base + 100
@sapient_energy_buffer_cap_t2 = 225   # 25 base + 200
@sapient_minerals_buffer_cap_t0 = 100 # 50 base + 50
@sapient_minerals_buffer_cap_t1 = 150 # 50 base + 100

# --- Purge windfall guard --------------------------------------------------
# The buffer treadmill measures NET income, windfall included — so a purge
# windfall (+~200 energy while grinding down captured pops) SATISFIES the
# buffer for free and the treadmill idles exactly while the ghost income
# lasts; when the purge ends, recurring costs (fleet upkeep) contracted
# against it crater the economy. The guard is a separate FLAT lane (not a cap
# bypass — the treadmill stays tech-capped): while a purge is running AND
# income is visibly windfall-inflated (>= floor), it adds a demand big enough
# to push the target ABOVE what the windfall satisfies. Deliberately
# unreachable while purging: it is pure build pressure that converts ghost
# income into real districts, and it self-cancels the moment the purge ends.
# THREE guards, keyed to the purge job that decides the windfall resource
# (see sapient_purge_windfall_* in 10_sapient_plan_triggers): machine purgers
# -> energy; organic purgers, non-lithoid prey -> food (food-fleet empires
# only); organic purgers, LITHOID prey -> minerals (minerals-fleet empires
# only). Shared floor + magnitude below; split per-resource if tuning
# demands it. Size guard >= typical windfall minus buffer, or it never bites.
@sapient_purge_windfall_floor = 100   # arm only when income >= base + ~75
@sapient_purge_windfall_guard = 150   # flat extra demand while armed

# =============================================================================
# WAR ESCALATION  (LIVE — one buildable unit per rung; see TUNING UNIT below)
# =============================================================================
# HISTORY: this began as inert scaffolding at 0; the rungs are now dialed to
# one increment each, so the "at 0 they're always fulfilled / inert and safe"
# analysis below NO LONGER APPLIES — these are real targets. They are small
# enough to stay reachable in practice (tested), and the fulfillment-gate
# coupling warning below turned out to be dead documentation anyway (observer-
# confirmed: treadmills stack fine with Base Pops unfulfilled).
# Subplan ai_weights are inert (vanilla 00_example.txt: "ai_weights are NOT added
# from subplans"). So the old `modifier = { add = 50  sapient_threat_* }` blocks
# did nothing. War escalation is re-expressed as REAL (non-scaling) subplans that
# add a BOUNDED amount on top of the peacetime target while a threat rung holds,
# then drop off when it clears. Not `scaling = yes`: a scaling war lane would
# re-add until its potential failed and could run away over a long war.
#
# GRADUATED + CUMULATIVE: the rungs nest (mobilizing -> alarmed -> wary), so at
# war all firing tiers sum. Reproduces the old +50/rung stacking intent.
#   peacetime target + wary + alarmed + mobilizing + atwar
#
# TUNING UNIT: one buildable increment from 05_sapient_job_output_reference —
#   alloys 3.75 (1 foundry zone)   energy 12 (1 rural district)
#   minerals 8 (1 rural district)  food 12 (1 rural district)
#   naval 5 (1 military zone)
# e.g. set _wary = one unit, _alarmed = two, etc., for a rising staircase.
#
# ⚠ COUPLING (verify in observer play): a non-optional subplan that is UNFULFILLED
#   appears to gate ALL scaling globally — vanilla marks its pops goal
#   `optional = yes` for exactly this reason. So keep war targets REACHABLE, or
#   add `optional = yes` to a war subplan to let it add demand without ever
#   freezing peacetime growth. At 0 they're always fulfilled (unless in deficit),
#   so the scaffolding is inert and safe until you dial a number up.
#
# ⚠ TIER COLLAPSE: sapient_threat_mobilizing currently == is_at_war (see
#   20_sapient_world_triggers.txt), so the mobilizing and atwar tiers fire
#   together until you split that trigger. The 4th tier is wired but redundant
#   with the 3rd for now — split mobilizing (imminent, pre-shooting) from
#   is_at_war (shooting) to make all four distinct.
