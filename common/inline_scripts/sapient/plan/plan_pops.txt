# SapientAI — pop demand
# TESTED FINDING (observer): the AI does not execute colony_automation or
# colony_automation_exceptions AT ALL — assembly buildings on AI worlds come
# from the HARDCODED builder. Hypothesis under test: the builder consumes the
# plans' pops goal for growth investment.
#
# Vanilla marks both pops goals `optional = yes` so they never gate scaling
# (a growth target that can't be met would otherwise freeze the whole economy).
# We keep them NON-optional here on purpose, to test whether an ungated pops
# target actually pressures the builder toward pop-growth buildings. If this
# freezes scaling for empires that can't hit the target, flip these back to
# `optional = yes`.
#
# Inert flat weights removed — they never did anything. Under treadmill
# semantics "Scaling Pops" targets current + step for every empire: pressure is
# relative, universal, and permanent.

}
