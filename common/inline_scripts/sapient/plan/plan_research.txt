# Research — base floor for all (three disciplines). Below the colony-split
# threshold ONE unified scaling lane moves all three in lockstep (keeps small
# empires on generic research designations); above it, per-discipline lanes let
# planets specialize.
#
# The old `add = 100 sapient_research_arc` weight was meant to boost research
# priority for arc empires (>=4 perks or servitor). Inert, and unnecessary:
# scaling lanes don't compete by weight, and unity scaling already self-stops at
# the perk cap (see plan_unity), so research naturally continues alone afterward.
# Weight removed. Note: research scaling has no income/stockpile cap of its own —
# it is bounded only by the "fulfilled" check. Flagged for a later pass if the
# late-game research treadmill needs its own ceiling.
# Base Research floor (all three disciplines) is UNCONDITIONAL, so it lives in the
# top-level income block of 01_base.txt (vanilla puts research there too). Only the
# scaling lanes remain here.
# Unified lane (<10 planets) scales at two rates by arc, mirroring unity. Only
# this lane is arc-tuned: at >=10 planets the split lanes below run all three
# disciplines at full step (triple demand), which already dominates.
#
# RESEARCH ARC (4th perk onward, or servitor): research is the priority — higher.
subplan = {
	scaling = yes
	set_name = "Scaling Research (Unified, Research Arc)"
	potential = {
		NOT = { sapient_research_split = yes }
		sapient_research_arc = yes
	}
	income = {
		physics_research = @sapient_research_step_unified_research_arc
		society_research = @sapient_research_step_unified_research_arc
		engineering_research = @sapient_research_step_unified_research_arc
	}
}
